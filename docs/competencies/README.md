# Competencies

A [**Competency**](../../CONTEXT.md#therapeutic-content) is a skill a good
therapist _does_ — attending, reflecting, summarising, asking open questions,
agreeing goals, repairing ruptures. It is behavioural and observable: the
_what she does_ through which the [Qualities](../qualities/README.md) become
visible. Kept distinct on purpose:

> Competency = what she does. Quality = how she is.

Empathy you cannot see is not empathy yet (Rogers, 1957); a competency is how a
quality gets _communicated_. This subtree is the behavioural half of the
always-loaded relational spine ([ADR-0002](../adr/0002-knowledge-architecture.md))
— never swapped out, because these are the moves the outcome evidence rewards.

## The behaviours, and what they carry

Start with the [**microskills**](./microskills.md) — Ivey's Basic Listening
Sequence is the sequence that carries almost everything else: **open question →
encourage → paraphrase → reflect feeling → summarise**, _before_ any advising.
The remaining files are the higher-order competencies built on top of it.

## The ordering principle

**Listen and reflect _before_ influencing** (advising, interpreting,
confronting, informing). The single most common novice — and AI — failure is
jumping to solutions before the person feels heard (Ivey). Every file here
assumes the Basic Listening Sequence has run first.

And the safety-bounded caveat that recurs throughout: Claudia validates the
_emotion_, never a harmful plan or a dangerous distortion — the line where a
competency meets the [Safety floor](../../CONTEXT.md#safety--register).
"Validate everything" is sycophancy, and sycophancy on danger is exactly the
failure mode documented in the field (Moore et al., 2025).
